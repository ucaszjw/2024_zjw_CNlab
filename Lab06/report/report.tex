\section{实验内容}

\begin{enumerate}
    \item 根据附件材料中提供的脚本，重现PPT中h1(发送方)在对h2进行iperf的同时测量h1的拥塞窗口值(cwnd)、r1-eth1的队列长度(qlen)、h1与h2间的往返延迟(rtt)的实验结果；
    \item 变化r1-eth1的队列大小，考察其对iperf吞吐率和上述三个指标的影响；
    \item 根据附件材料中提供的脚本，重现PPT中Tail Drop、RED、CoDel三种队列管理算法的实验结果。
\end{enumerate}

\section{实验结果与分析}

\subsection{重现BufferBloat实验}

取\texttt{maxq}为100，在终端中输入：

\begin{lstlisting}[language=bash]
    sudo python3 reproduce_bufferbloat.py --maxq 100
\end{lstlisting}

程序会输出三个.txt文件，我使用了python脚本读取这三个文件中的内容，获得相应数据，并输出所绘制曲线图。重现实验结果如下：

\begin{enumerate}
    \item CWND结果
    
    \begin{figure}[H]
        \centering
        \includegraphics[width=0.55\textwidth]{cwnd_reproduce_plot.png}
        \caption{CWND重现结果}
    \end{figure}

    与PPT中的结果十分一致。

    \item QLEN结果
    
    \begin{figure}[H]
        \centering
        \includegraphics[width=0.55\textwidth]{qlen_reproduce_plot.png}
        \caption{QLEN重现结果}
    \end{figure}

    与PPT中的结果十分一致。

    \item RTT结果
    
    \begin{figure}[H]
        \centering
        \includegraphics[width=0.55\textwidth]{rtt_reproduce_plot.png}
        \caption{RTT重现结果}
    \end{figure}

    与PPT中的曲线走势较为一致，均有显著周期性的上升下降趋势。但由于相邻ping程序间隔会发生变动，导致实验结果与PPT中的结果有一定差异。
\end{enumerate}

\subsection{变化队列大小对iperf吞吐率和指标的影响}

分别取\texttt{maxq}为100、80、60、40、20，实验结果如下：

\begin{enumerate}
    \item CWND结果
    
    \begin{figure}[H]
        \centering
        \includegraphics[width=0.55\textwidth]{cwnd_changemaxq_plot.png}
        \caption{CWND变化maxq结果}
    \end{figure}

    刚开始测试时，拥塞窗口达到峰值，随后迅速回落至队列大小相近水平，随时间推移，拥塞窗口逐渐增大，接着随着丢包，拥塞窗口迅速回落，循环往复。

    \item QLEN结果
    
    \begin{figure}[H]
        \centering
        \includegraphics[width=0.55\textwidth]{qlen_changemaxq_plot.png}
        \caption{QLEN变化maxq结果}
    \end{figure}

    同样，刚开始接收到数据包，队列长度迅速增大，当队列长度达到最大值时，开始丢包，队列长度迅速回落，循环往复。随着\texttt{maxq}增大，队列长度的峰值逐渐增大，丢包周期和变化幅度也逐渐增大，BufferBloat现象更加明显。

    \item RTT结果
    
    \begin{figure}[H]
        \centering
        \includegraphics[width=0.55\textwidth]{rtt_changemaxq_plot.png}
        \caption{RTT变化maxq结果}
    \end{figure}

    RTT随着队列长度的变化，也呈现出周期性的上升下降趋势，且与队列长度的变化趋势一致。随着\texttt{maxq}增大，RTT的峰值逐渐增大，变化幅度也逐渐增大。由于ping程序输出不稳定，图中曲线可能出现数据丢失（如图中maxq为20时的水平线段）。

    \item iperf吞吐率结果
    
    \begin{figure}[H]
        \centering
        \includegraphics[width=0.65\textwidth]{bandwidth_over_time.png}
        \caption{iperf吞吐率变化maxq结果}
    \end{figure}

    刚开始接收到数据包，iperf吞吐率也迅速增大，之后迅速回落，稳定在10Mbps左右，这正是链路的带宽。此外也观察到吞吐率会有周期性的向下波动，这是由于队列长度的变化导致的丢包，从而导致iperf吞吐率的波动。
\end{enumerate}

\subsection{重现队列管理算法实验}

分别使用Tail Drop、RED、CoDel三种队列管理算法，在终端中输入：

\begin{lstlisting}[language=bash]
    sudo python3 mitigate_bufferbloat.py -a taildrop # or -a red or -a codel
\end{lstlisting}

实验结果如下：

\begin{figure}[H]
    \centering
    \includegraphics[width=0.65\textwidth]{rtt_mitigate_plot.png}
    \caption{RTT重现队列管理算法实验结果}
\end{figure}

可以看出，Tail Drop算法的RTT显著高于RED和CoDel算法，且CoDel算法的RTT表现更佳。而且Tail Drop曲线出现了较稳定峰值，说明实验较为准确。

\section{实验总结}

通过本次实验，我成功重现了BufferBloat实验，并且观察到了队列大小对iperf吞吐率和指标的影响，这让我更加深入地理解了数据包对了的工作原理和BufferBloat现象。此外，我还重现了Tail Drop、RED、CoDel三种队列管理算法的实验，发现CoDel算法的RTT表现更佳，这也让我对队列管理算法有了更深入的了解。
